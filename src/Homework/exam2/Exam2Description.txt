Курсовая по многопоточности.

В программе присутствуют:
    - клиент;
    - сервер.

Порядок действий:
1. Клиент устанавливает соединение с сервером.
2. Пользователь вводит сообщение в консоль.
3. Клиент формирует объект Message и отправляет его на сервер

Поля Message:
    - дата отправки;
    - отправитель;
    - текст сообщения.
+ у сообщения могут быть дополнительные поля, если это требуется.

4. Сервер рассылает полученное сообщение всем подключенным клиентам,
кроме отправителя (задача на тренировку очереди).
5. Клиент выводит полученное сообщение в консоль

На клиенте должно быть два потока: основной и дополнительный.
Первый поток взаимодействует с пользователем:
    - читает сообщения из консоли, формирует объекты Сообщений
    и отправляет на сервер;
Второй поток:
    - получает входящие сообщения, выводит их в консоль.

Сервер:
Серверу необходимо количество клиентов (отдельных подключений) - Set или Map.
И Set, и Map должны быть потокобезопасны.
На каждое подключение создается поток - Reader - который читает
сообщения от клиента.
Всего один пишущий поток - Writer - который рассылает сообщения клиентам.
(поток взял сообщение - переложил его в хранилище. Какое - необходимо выбрать)
((но очередь удобнее всего))
Поток Writer будет брать сообщение из очереди и рассылать всем клиентам,
хранящимся в Set или Map.
Клиент отсоединился? Необходимо удалить клиента из коллекции.
Его личный Reader должен закрыться.
